Build a full-stack web application called **“File System Recovery & Optimization Simulator”**.

---

## 🎯 Objective:

Design and implement a **simulated file system environment** that demonstrates:

* File storage mechanisms
* Free-space management
* File access methods
* Disk crash scenarios
* Recovery techniques
* File system optimization

⚠️ Important: This is a **simulation tool**, not real disk-level recovery. All operations should be modeled logically within the application.

---

## 🧩 Core Features:

### 1. 📁 File System Simulation

* Create a virtual file system:

  * Root directory with nested folders and files
* Each file should have:

  * Name, size, type, creation date
  * Simulated disk block allocation
* Display structure in:

  * Tree view (like file explorer)

---

### 2. 💾 Free Space Management

* Simulate disk blocks (e.g., 100 blocks)
* Implement:

  * Bitmap method (0 = free, 1 = used)
* Visualize:

  * Disk block allocation
* Show how files occupy blocks

---

### 3. 📂 File Access Methods

* Simulate:

  * Sequential access
  * Direct access
* Show:

  * Time taken for read/write operations (simulated metrics)

---

### 4. 💥 Disk Crash Simulation

* Add controls:

  * “Simulate Crash”
  * “Corrupt File”
* Effects:

  * Randomly mark files as deleted/corrupted
  * Remove metadata partially

---

### 5. 🔄 Recovery Mechanism

* Implement recovery techniques:

  * Journaling / log-based recovery
  * Backup metadata restoration
* Show:

  * Recovery status (success / partial / failed)
  * Recovery probability score

---

### 6. ⚡ Optimization Module

* Simulate:

  * Defragmentation
  * Block reallocation
* Show:

  * Before vs After performance
  * Reduced access time

---

## 🖥️ Frontend Requirements:

* Use **React + Tailwind CSS**
* UI Components:

  * Sidebar (File System / Recovery / Optimization)
  * Tree view for directories
  * Disk block visualization grid
  * Dashboard cards (stats, performance)
* Add:

  * Animations (for scanning, recovery)
  * Progress bars
  * Toast notifications

---

## ⚙️ Backend Requirements:

* Use **Node.js + Express**
* API Endpoints:

  * `/create-file`
  * `/delete-file`
  * `/simulate-crash`
  * `/recover-files`
  * `/optimize-system`
* Maintain:

  * In-memory data structure representing file system
* Implement logic for:

  * Block allocation
  * Recovery algorithms
  * Performance calculation

---

## 📊 Visualization (Important):

* Disk blocks grid (highlight used/free)
* File fragmentation view
* Recovery logs panel
* Performance comparison charts

Create a **clean, interactive educational tool** that demonstrates how operating systems handle file storage, recovery, and optimization in real-world scenarios through simulation.
